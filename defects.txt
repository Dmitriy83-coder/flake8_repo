# Баг-репорты для класса StringUtils

## 1. Проблема с методом `trim`
### Описание
Метод `trim` не удаляет пробелы в конце строки.

### Шаги для воспроизведения
1. Вызвать `StringUtils.trim("   hello world   ")`
2. Ожидалось: "hello world"
3. Получено: "hello world   "

### Ожидаемое поведение
Метод должен удалять пробелы как в начале, так и в конце строки.

---

## 2. Проблема с методом `contains`
### Описание
Метод `contains` не учитывает регистр символов.

### Шаги для воспроизведения
1. Вызвать `StringUtils.contains("SkyPro", "s")`
2. Ожидалось: False
3. Получено: True

### Ожидаемое поведение
Метод должен возвращать `False`, если символ не найден с учетом регистра.

---

## 3. Проблема с методом `delete_symbol`
### Описание
Метод `delete_symbol` не удаляет все вхождения символа, если символ является подстрокой.

### Шаги для воспроизведения
1. Вызвать `StringUtils.delete_symbol("SkyPro", "Pro")`
2. Ожидалось: "Sky"
3. Получено: "Sky"

### Ожидаемое поведение
Метод должен удалять все вхождения подстроки, а не только первое.

---

## 4. Проблема с методом `capitalize`
### Описание
Метод `capitalize` не обрабатывает строки, состоящие только из пробелов.

### Шаги для воспроизведения
1. Вызвать `StringUtils.capitalize("   ")`
2. Ожидалось: "   "
3. Получено: "   "

### Ожидаемое поведение
Метод должен возвращать строку без изменений, если она состоит только из пробелов.

---

## 5. Проблема с методом `delete_symbol`
### Описание
Метод `delete_symbol` не обрабатывает случай, когда передан пустой символ.

### Шаги для воспроизведения
1. Вызвать `StringUtils.delete_symbol("SkyPro", "")`
2. Ожидалось: "SkyPro"
3. Получено: "SkyPro"

### Ожидаемое поведение
Метод должен возвращать строку без изменений, если символ для удаления пустой.
